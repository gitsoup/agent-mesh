# /align

Align a plan with repo context and decisions.

## Steps

1. Determine affected context from `CONTEXT-MAP.md`.
2. Read relevant context files and ADRs.
3. Identify ambiguous or conflicting terms.
4. Challenge the plan against existing code and constraints.
5. Ask decision-shaping questions only when needed.
6. Recommend defaults and update context if terminology is clarified.
7. Create an ADR only for durable decisions.
8. Output an alignment summary.
